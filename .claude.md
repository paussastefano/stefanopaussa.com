# Stefano Paussa Portfolio Website

## Project Overview

Portfolio website for Stefano Paussa, a videographer and editor based in Milano, Italy. The site showcases his work across editing, videography, and photography, providing a minimal and elegant platform for potential clients and collaborators.

## Goals & Objectives

1. **Professional Portfolio**: Clean, minimal showcase of Stefano's video and photography work
2. **Easy Content Management**: Enable Stefano to add/edit projects without coding knowledge
3. **Fast & Performant**: Static site with optimized media loading
4. **Mobile Responsive**: Works seamlessly on all devices
5. **Future-Ready**: Prepared for migration to Hugo + Decap CMS for advanced content management

## Current Technology Stack

- **HTML/CSS/JavaScript**: Static frontend
- **JSON-based Content**: `data/projects.json` stores all project data
- **GitHub Pages**: Free hosting and deployment
- **Custom Fonts**: Alte Haas Grotesk (Regular & Bold)
- **Video Embedding**: YouTube embeds + local video files

## Key Features Implemented

### 1. Landing Page (index.html)
- Local video (PARKOUR.mp4) plays on user interaction (Play button)
- Audio enabled after user click
- Automatic redirect to home.html when video ends
- Full-screen cover video

### 2. Home Page (home.html)
- YouTube video background (looping)
- Minimal navigation header overlay (white text on video)
- Navigation: Stefano Paussa, Works, Photography, About

### 3. Unified Navigation
- Consistent header across all pages
- Active page underlined
- Same structure as home but with black text on white background

### 4. Works Page (works.html)
- Dynamic project loading from JSON
- Filter buttons: All, Editing, Videography
- 2-column grid of project cards
- Each card shows: thumbnail, title, role
- Clicking opens work-detail.html with project ID

### 5. Photography Page (photography.html)
- 2-column grid of photography images
- Loads photography projects from JSON
- Clean scroll layout

### 6. Work Detail Page (work-detail.html)
- Single project view loaded dynamically via URL parameter (?id=project-id)
- Video embed or image display
- Project description
- Full credits list
- Back to Works link

### 7. About Page (about.html)
- Single paragraph bio centered vertically
- Max-width: 615px for optimal readability
- Email link: paussastefano@gmail.com (mailto, underlined, black)
- Minimal layout matching site aesthetic

## Design Principles

- **Minimal**: Clean, no unnecessary elements
- **Typography**: Alte Haas Grotesk font throughout
- **Colors**: Black text on white (pages), white text on video (home)
- **Spacing**: Generous whitespace, aligned to header padding
- **Responsive**: Works on desktop and mobile
- **No Emojis**: Professional, text-only approach

## Content Management (Current System)

### Adding a New Project

Edit `data/projects.json` and add:

```json
{
  "id": "unique-project-id",
  "title": "Project Name",
  "role": "Editor / Director / Photographer",
  "category": "editing|videography|photography",
  "thumbnail": "images/projects/thumb.jpg",
  "videoUrl": "https://www.youtube.com/embed/VIDEO_ID",
  "description": "Project description...",
  "credits": [
    { "role": "Director", "name": "Name Surname" },
    { "role": "Editor", "name": "Stefano Paussa" }
  ]
}
```

### Image Organization

## Future Migration: Hugo + Decap CMS

### Why Hugo + Decap CMS?

**Current Limitations:**
- Stefano must edit JSON manually
- No visual interface for content management
- Requires GitHub knowledge

**Future Benefits:**
1. **Web-based CMS**: Stefano accesses `/admin` to manage content
2. **Visual Editor**: Form-based project creation with drag-drop image uploads
3. **No Coding Required**: Add/edit/delete projects via UI
4. **GitHub Integration**: Decap CMS commits to GitHub automatically
5. **Hugo Speed**: Lightning-fast static site generation

### Migration Plan (Not Yet Started)

1. **Install Hugo** locally
2. **Convert HTML to Hugo templates**:
   - `layouts/_default/baseof.html` (base template)
   - `layouts/index.html` (home)
   - `layouts/works/list.html` (works grid)
   - `layouts/works/single.html` (project detail)
3. **Convert JSON to Markdown**:
   - Each project becomes `content/works/project-name.md`
4. **Setup Decap CMS**:
   - Add `static/admin/config.yml`
   - Configure collections for projects
   - Enable GitHub authentication
5. **Configure GitHub Actions** for auto-deployment
6. **Test locally**, then deploy

### Decap CMS Interface (Future)

Stefano will:
1. Go to `stefanopaussa.com/admin`
2. Login with GitHub
3. Click "New Project"
4. Fill form: title, role, category, upload thumbnail, paste video URL, write description, add credits
5. Click "Publish"
6. Site auto-rebuilds and deploys
